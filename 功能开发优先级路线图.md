# Quant Skill Vault 功能开发优先级路线图（当前阶段）

## 1. 阶段目标
- 当前阶段以功能增长为第一优先，不以用户系统/组织级治理为阻塞条件。
- 目标是在最短周期内提升“可复用、可管理、可回滚”的核心产品能力。

## 2. 优先开发功能（8项）
以下为建议优先级（从高到低）：

1. Skill 版本历史 + 一键回滚  
- 价值：防止误改、支持快速恢复。  
- 核心交付：版本列表、版本 diff、回滚按钮、回滚审计记录。  
- 验收标准：任意 Skill 可查看最近 N 个版本并一键恢复成功。

2. 模板中心（Template）  
- 价值：减少重复创建，提升团队产能。  
- 核心交付：模板创建、模板套用、模板更新。  
- 验收标准：新建 Skill 时可从模板一键填充核心字段。

3. 草稿管理页（Draft Manager）  
- 价值：集中管理草稿，避免“草稿丢失感”。  
- 核心交付：草稿列表、继续编辑、删除草稿、冲突标记。  
- 验收标准：用户可在独立页面恢复任意未完成草稿。

4. 发布流（Draft -> Published）  
- 价值：区分编辑态与稳定态，降低误用风险。  
- 核心交付：状态字段、发布动作、发布时间与发布人记录。  
- 验收标准：仅 Published 版本可作为推荐输出和对外导出默认版本。

5. 技能质量评分（Quality Score）  
- 价值：推动团队规范，形成可量化质量标准。  
- 核心交付：评分规则（tests/triggers/guardrails/完整度），页面展示。  
- 验收标准：每个 Skill 展示评分和提升建议。

6. AI 生成结果可追溯（Proposal Trace）  
- 价值：提升可解释性与复盘能力。  
- 核心交付：记录 AI 提案、采纳内容、最终变更。  
- 验收标准：可追踪“AI 建议 -> 用户采纳 -> 最终字段/文件结果”。

7. 导入能力（md/json/zip）  
- 价值：降低历史资产迁移成本。  
- 核心交付：导入向导、字段映射、冲突处理。  
- 验收标准：成功导入后可直接进入编辑与校验流程。

8. 使用统计（Usage Analytics）  
- 价值：数据驱动优化，识别高价值/高失败技能。  
- 核心交付：调用量、失败率、最近活跃趋势。  
- 验收标准：看板可查看 top skills 和失败热点。

## 3. 建议实施节奏
## Sprint 1（先打底）
- 版本历史 + 回滚
- 草稿管理页
- 发布流（最小可用）

## Sprint 2（提升复用与质量）
- 模板中心
- 质量评分
- AI 提案可追溯

## Sprint 3（扩展输入与运营）
- 导入能力
- 使用统计

## 4. 技术落地顺序（建议）
1. 先补数据模型（Version / PublishStatus / Template / ProposalLog / UsageEvent）。
2. 再补 API 层（列表、创建、回滚、发布、导入、统计）。
3. 最后补页面层与交互（列表、详情、管理页、看板）。

## 5. 当前执行决策
- 本文档作为当前阶段功能开发主清单。  
- 用户系统、SSO、权限治理暂缓，不阻塞上述 8 项功能推进。  
- 每完成一个功能，补一条“验收截图 + 回归测试记录”。
