# Quant Skill Vault 用户体验改进存档（暂缓项）

## 1. 存档目的
- 记录会影响用户体验但当前阶段暂不优先开发的改进点。
- 作为后续稳定性与体验专项的输入清单，避免需求遗失。

## 2. 暂缓的体验改进项
1. 启动链路稳定化（DB/环境变量/一键启动检查）  
- 现象：环境不完整时可能出现接口 500。  
- 建议：增加 `preflight` 检查脚本和启动诊断页面。

2. 草稿冲突的人工决策界面  
- 现状：冲突后自动同步最新版本。  
- 建议：增加“保留本地副本 / 使用云端版本”双分支选择。

3. 离开页面保护（未保存拦截）  
- 现象：误关闭或跳转可能造成编辑中断。  
- 建议：未保存变更时给出统一确认提示。

4. 统一错误码与错误文案策略  
- 现状：多接口错误表达不完全一致。  
- 建议：统一 error shape 与文案映射，提高可理解性。

5. 幂等与重试策略增强  
- 现状：网络抖动时可能重复操作。  
- 建议：关键写操作引入 idempotency key 或去重机制。

6. 长任务进度可视化（AI/导出/批量操作）  
- 现象：任务执行中用户不确定是否卡住。  
- 建议：进度条 + 可取消 + 完成通知。

7. 空态与异常态页面统一  
- 现状：空列表、失败页样式和引导不一致。  
- 建议：统一空态组件与错误重试入口。

8. 关键路径 E2E 扩容  
- 现状：已有测试，但关键业务回归仍可加强。  
- 建议：优先覆盖创建/编辑/草稿恢复/导出/批量操作主路径。

## 3. 触发升级条件（何时从“暂缓”转“执行”）
- 内部活跃用户明显增加（如 >15 人）；
- 线上问题集中在“稳定性/误操作/恢复困难”；
- 功能增长阶段完成，进入质量优化阶段。

## 4. 执行建议
- 每个功能开发 Sprint 留出 10% 容量用于修复本清单中的高影响问题。
- 一旦出现重复性故障，立即把对应条目提级为当前 Sprint 必做项。
